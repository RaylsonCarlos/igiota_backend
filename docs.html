  • User
    o que? Método Link Cabeçalhos campos resposta
    Criar / Modificar usuário POST /users Content-Type: application/json
    • name - text
    • email - text
    • active - boolean
    • password - text
    • confirm_password - text
    • phone - text

      • parâmetros adicionais para modificar um usuário:

    • userid - int
    • token - text
      JSON com:
    • id
    • email
    • name
    • phone
    Listar usuários GET /users - - JSON com lista de usuários
    Fazer Login POST /login Content-Type: application/json
    • email
    • password
      JSON com:
    • userId
    • token
    • name
    • valid_untill
    • login_success
  • Dividas
    o que? Método Link Cabeçalhos campos resposta
    Criar dívida POST /debts Content-Type: application/json
    • owner - int (userId)
    • owe_to - int (userId)
    • value - double
    • active - boolean
    • token - text
    		 JSON com a dívida recém criada
    Consultar dívidas por usuário (que possui com outrem ou que outrem possui com ele) GET /debts/'id do usuário' Content-Type: application/x-www-form-urlencoded - JSON com lista de dívidas
  • Pagamentos
    o que? Método Link Cabeçalhos campos resposta
    Criar pagamento POST /payments Content-Type: application/json
    • user - int (userId)
    • debt - int (debtId)
    • value - double
    • token - text
    		 JSON com o pagamento recém criado
    Consultar os pagamentos realizados sobre uma dívida GET /payments/'id da dívida' Content-Type: application/x-www-form-urlencoded - JSON com lista de pagamentos
